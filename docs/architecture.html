POST/api/auth/register
註冊新帳號，bcrypt 雜湊密碼，產生 SHA-256 shareCode
Body: { username, password, nickname }
200: { id, username, nickname, shareCode }
GETPOST/api/auth/[...nextauth]
NextAuth 通用處理器 (登入 / 登出 / session / callback)
策略: Credentials Provider + JWT session
GET/api/songs?schema=v2
取得全部歌曲 (含 members[].id + units)。Schema 仍在迭代，所以不走快取，每次直出。
200: Song[] (含 members + units)
Cache: no-store
GET/api/songs/[id]/users
某首歌「會唱／常聽」的公開使用者清單
條件: familiarity ∈ {1,2} AND user.isPublic = true
200: { nickname, themeColor, familiarity }[]
GET/api/selectionsAUTH
取得目前使用者所有熟悉度標記 (轉成 map)
200: { songId: familiarity, ... }
401: 未登入
POST/api/selectionsAUTH
批次 upsert / delete (familiarity=0 即刪除)。用 Prisma transaction 保證原子性。
Body: [{ songId, familiarity }, ...]
200: { success: true }
POST/api/collab
多人歌單聯集：取得 N 個 shareCode 任一人標記過的歌 (前端再用 chip 二次篩選)
Body: { shareCodes: string[] }（至少 2 個）
200: { users: nickname[], songs: Song & { ratings } }
GET/api/user/search?q=
依 nickname 模糊搜尋公開使用者 (大小寫通用，上限 50 筆；空 q 回所有公開使用者)
Query: ?q=P名 (可空)
200: { nickname, shareCode, themeColor }[]
POST/api/user/settingsAUTH
更新 P名、主題色 (#RRGGBB)、是否公開
Body: { nickname, themeColor, isPublic }
200: 更新後的欄位
GET/api/colors
讀取靜態 idol-colors.json (偶像代表色清單)
200: { name, color }[]
